java基础 （HeadFristJava笔记）

java的基础知识点整理。👱

第1~2章：类与对象:

定义：类是有相同特征的东西的集合，

  • 类是对象的蓝图。
  • 对象是已知的事物，对象是类的具体体现，举例来说对象就好比是电话薄里面的一比数据
  • 对象本身已知的事物被称为“实例变量”
  • 对象可以执行的动作被称为 方法

第3~4章：变量和对象的行为

  • 变量：变量分为primitive主数据类型 和引用数据类型。
    例如int，double，等基本数据类型就是主数据类型，(整型：byte,int,short,long
    浮点型:double,float,字符型: char布尔型:boolean)例如我有一个Dog类，我生成一个myDog对象，Dog就是引用数据类型，

  • 对象的行为 ：对象有 状态和行为 两种属性，分别由实例变量 和 方法来表示，类定义对象的所知和所为，对象的所知是 实例变量，对象的所为是 方法。
    实例变量是声明在类中的 实例变量在初始化时就算你不给他赋值或者没有调用set方法，它也会有默认值，int -0，boolean -false，references -null,局部变量是声明在方法中中的，局部变量在使用前是必须被初始化的。

  • 对象的声明，创建，赋值:
    1. 对象类型是primitive主数据类型 EG： byte x = 7解释为代表数值7的字节被放进名称为x的变量中。
    2. 对象类型是引用变量类型，EG：Dog myDog = new Dog(); 解释为声明一个引用变量，他的类型是被永远固定为Dog类型，并将此变量命名为myDog,并将在推空间上新创建的Dog对象赋值给myDog这个引用变量。

第5~6章 ：超强力方法和使用java函数库

  • 在java的API中，类是被包装在包中，要使用API中的类，你必须知道它被放在哪个包中

  • 将String类型的变量Guess转换成int类型的变量的方法 int guess = Integer.parstInt(Guess)

  • 加强版的for循环(针对数组或者集合) for(String name ：nameArray){}

  • .equal(),判断对象是否于方法的参数相等，Math.random()用于产生随机数，等等

  • ArrayList是java API的类，继承自java.util类，是java函数库中数百个类中的一个，ArrayList类里面有很多方法，例如add(Object elem)向list中加入对象参数,remove(int index)移除此列表中指定位置上的元素，详细见–jdk文档.
    ArrayList的声明例子ArrayList<Egg> myList = new ArrayList<Egg>()翻译过来是创建一个Egg类型的list，它的名称叫myList,

  • import相当于节省了调用javaAPI的包名代码，例如import java.util.ArrayList和不加importjava.util.ArrayList<Dog> list = new java.util.ArrayList<Dog>()是一样的的，如果不加import就不管在声明，用作参数都需要加上包名，加import就是为了简化代码。
  • 布尔表达式:“与”和“或”运算符(&&,||), 不等于运算符(!=,!)

第7~8章：继承与多态

重要概念：继承，重写(发生在继承形态)，重载(发生在一个类中)，多态，抽象类，抽象方法，OBjiect类，接口。

  • 继承的意义：减少重复代码，增强代码的可维护性，
  • 调用一个继承类的方法时，先从最底阶开始找，先找子类有没有这个方法，再找父类。
  • 判断某物是否应该去继承某一物，用IS-A测试。

  • 多态：在多态下，引用与对象可以是不同的类型，例如Animal mydog = new Dow();其中dog是animal是子类。不仅仅是引用变量类型可以多态，参数和返回类型更是可以多态

  • 多态的Demo:

    
//假设Dog类，Hippo类都是Anima的子类
class  Vet{
//giveShot()这个方法可以取用任何一种的Anima，只要传入的是Anima的子类它都能执行，
 public void giveShot(Anima a){   
     a.makeNoise();
 }
}
class PetOwner{
 public void start(){
     Vet v = new vet();
     Dog d = new Dog();     
     Hippo h = new Hippo();
     v.giveShot(d);           //会执行Dog的makeNoise();
     v.giveShot(h);              //会执行Hippo的makeNoise();
 }
}

  • 覆盖(重写)是发生在子类与父类的一种行为，对父类的方法不满意时就可以修改，但修改的同时需要遵守2个约定：1.覆盖后的方法参数必须和父类的参数一样，且返回类型必须要兼容。2.不能降低方法的存取权限，举例你不能覆盖掉一个共有的方法并且把它修改为私有。

  • Object类是java中所有类的父类，Object类主要有两个主要目的，其一是作为多态让方法可以应付多种类型的机制，其二是提供java在执行期间对任何对象都有需要的方法的实现程序代码。
  • 接口：需要用到多重继承时会用到接口。使用接口就可以继承超过一个以上的来源，JAVA的接口就好像时100%的纯抽象类，
    接口的注意要点：
  1. 接口必须有子类，并且子类要实现全部的抽象方法。
  2. 一个接口可以继承多个接口(多重继承)。
  3. 一个子类可以继承抽象类并且实现接口。
  4. 以interface这个关键字取代class来声明接口。
  • 接口和抽象类的区别：
    1. 抽象类中除了有抽象方法，也可以有普通方法，可以有方法体，接口中没有体
    2. 接口并不属于类的继承体系，抽象类则属于，抽象类一定是作为父类存在，但是，就算毫无关系的两个类也可以实现同一个接口，接口的出现就是为了解决多重继承的问题，又叫“致命方块”—-出自HeadFristJava,P223页
    3. 一个子类只能继承一个抽象类，但是却可以同时实现多个接口。

第9章：构造器和垃圾回收器

重要概念：堆(heap)和栈(stack),构造器，对象的生命周期,垃圾收集器GC，super和this，

  • 栈(stack)：一种先进后出的线性结构，和队列(先进先出)一样都是重要的线性结构表，在java中栈空间主要来保存方法调用和局部变量
  • 堆(heap):一种可垃圾回收的生存空间，所有的对象都存活在堆上

  • 构造器：构造器也叫构造函数或者构造方法，是一种和类同名，并且没用返回值的方法。

    构造器带有你在初始化对象时会执行的程序代码。也就是新建一个对象会被执行。就算你没有自己写一个构造器，编译器也会帮你写一个无参的。

    构造器的关键作用：

    一. 能在对象能够被赋值给引用之前就执行。

       public class Duck{
          public Duck(){System.out.println("Quack");}

      public class UseDuck{
      public static void main(String[] args){
          Duck d = new Duck();
      }
      }    输出Quack

    二. 使用构造函数来初始化对象的状态

    所以根据这个特点可以知道构造函数可以重载，也就是说你可以拥有一个以上的构造函数，但是有个约定，参数需要不同，如果参数相同，参数的顺序不同也是可以的。

  • 父类以及继承与构造器之间的关系：简单的说构造器能继承吗？答案是能，但是又比一般的继承多一点东西，在创建新对象时，所有继承下来的构造器都会执行，直到执行到Object这个类为止(我们都知道Object是祖先类)

  • super()与super.的区别：super关键字是用来调用父类的方法的，而super()是调用父类的构造器，super.是调用父类的一般方法。注意对super()的调用必须是构造函数的第一条语句

  • this关键字是对对象本身的引用，有以下几个用法：

    1. 使用this()来从某个构造器调用同一个类的另外一个构造器，例如this();调用默认的构造器，this(参数)调用带参数的构造器。
    2. 限定当前对象的数据域变量。一般用于方法内的局部变量与对象的数据域变量同名的情况。如 this.num = num。this.num 表示当前对象的数据域变量 num，而 num 表示方法中的局部变量。
    3. 注意this()只能用在构造器中，且必须是第一句，

  • super()与this()不能同时出现，

  • 对象有以下3种情况会被GC回收：

    1. 引用变量永久离开对象的范围 
      public class StackRef{
      public void foof(){
      barf();
      }
      public void barf(){
      Duck d = new Duck();
      }
      }
    • 引用被赋值到其他的对象上
      public class ReRef{
      Duck d = new Duck();
      public void go(){
      d = new Duck();
      }
      }
      新的duck会放在堆上，只要ReRe活着，d就没事，但是调用了go();后，d被引用到了其他的Duck，第一个Duck就等着垃圾回收了。

  • 直接将引用设定为null，

    public class ReRef{
    Duck d = new Duck;
    public void go(){
    d = null;
    }
    }
    d被设定null后，就好像没用操作对象的遥控器一样。
